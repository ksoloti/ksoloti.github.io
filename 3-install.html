3. Install

Update: a standalone patcher for Ksoloti is currently being developed. You don't need to follow any of the below installation guides (3.1, 3.2, 3.3). You can download the beta for your OS here.

Extract the package wherever you like and run the launcher file (Ksoloti.exe, Ksoloti.app, Ksoloti ...). Nothing will happen for a while until the Patcher has figured out who and where it is and has downloaded the object libraries to your OSes user home folder.

The first time you connect your Ksoloti you will probably have to flash it with the standalone firmware using Rescue Mode. Before powering up your Ksoloti, press and hold S1, then connect USB. The Ksoloti's LEDs will stay off. In the standalone Patcher, select Board > Firmware > Flash (Rescue) and wait for it to finish. If you're on Windows and the Ksoloti in rescue mode is not recognized, check out how to install a working bootloader driver using Zadig in this guide.

Did something in the firmware update go wrong? Does your Ksoloti appear "bricked"? Its LEDs do not light up when you power it up, or they do light up but the Patcher cannot detect it? Rescue Mode is here to save the day!
